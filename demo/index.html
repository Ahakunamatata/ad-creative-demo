Project Hub

先从项目入口开始，不把 Intake、策略、Storyboard 挤在同一页。

这版原型改成了真实的多页面流。用户先在这里看到历史项目、状态和下一步动作，再进入引导式创建、策略选择和核心工作区。

为什么这样改

原来的单页 demo 更像“功能陈列”。多页面后，交互逻辑会更像真实产品：项目入口 -> brief 生成 -> 策略选择 -> storyboard 生产。

Flow

推荐工作流

1. 任务 Intake

先录入产品与 brief，不直接写长表单。支持粘贴商店链接，系统优先自动识别产品信息。

2. 策略选择

先选脚本路线，再决定哪条值得进入分镜生产，而不是一上来就直接生成整条视频。

3. Storyboard 工作区

左侧管镜头脚本，中间预览视频结构，底部排时间线，右侧调字幕、多语言和声音。

线框级原型，当前重点是对象、页面和交互，不是正式 UI。
Create

从哪里开始新项目？

这一步不直接让用户填一大堆字段，而是先告诉用户接下来会走一个有引导的创建流程：识别产品信息 -> 选择模板 -> 补全简报 -> 生成策略。

方式 A：从商店链接开始

粘贴 App Store / Google Play 链接，系统优先自动识别 icon、名称、描述、痛点和承诺，减少用户组织语言的负担。

方式 B：手动输入

如果是内部测试包或没有公开商店页，也能手动补充产品信息，并继续走模板化的 brief 流程。
